    Oct 15, 2008 05:57 PM UTC by Terrence Martin 
    The problem was that the CEMON tomcat-55 needed to be restarted. It last
ran on the 10th of October. It can be a bit fiddly, although I have not
had any issues for a while.

It is possible that tomcat-55 halted due to memory starvation system
wide. The system has 4GB of RAM but only 2GB of swap. You may wish to
consider adding a large swap file to augment the virtual memory. 12GB
should be fine. This is as efficient as swap partitions in modern linux.

To create a linux swap file pic a directory that is not removed
automatically and run

dd if=/dev/zero of=swapfile-1 bs=1024 count=12000000

That will create a 12GB file with zeros

Then

mkswap swapfile-1
swapon swapfile-1

Then in /etc/rc.local add

swapon /dir/to/swap/swapfile-1

Terrence
