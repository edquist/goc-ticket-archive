-- by Douglas Strain
    May 25, 2012 08:51 PM UTC 
    The schedd error message about not being able to find the address of the startd running on the schedd machine is likely from an obscure feature of the schedd: if it is not contacted by the negotiator for a long time, it will attempt to directly claim the startd (if there is one) on the same machine as the schedd, without going through negotiation.  The error message about not being able to find the startd address is expected in the case where there is no local startd.

--Dan Bradley

Christophe, I think the SBGrid collector needs to be able to talk to your schedd, or possibly to your collector, which relays the info about your schedd.  The schedd will listen on different ports, but register itself with the collector.
You can start by opening tcp port 9618, which is the default Collector port (assuming you haven't customized it)
You probably also need a small range in the ephemeral port range as mentioned below, udp and tcp.
